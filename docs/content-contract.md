# Content Contract

## Purpose

This document defines the minimum required UI and copy structure for v1 so future agents do not invent product or layout content unnecessarily.

## Brand Header

`Locked`:

- logo
- lightweight navigation
- one small utility area

`Preferred`:

- nav labels: `Shop`, `Story`, `Freshness`
- utility item: simple cart or bag icon only

The header should feel like quiet brand chrome, not an app dashboard.

## Hero Copy

`Locked`:

- one short headline
- one CTA
- optional small supporting metric row

`Preferred` placeholder direction:

- headline tone: editorial, warm, concise
- CTA tone: invitational rather than transactional

Example placeholder direction:

- headline: `Rich & Aromatic Lounge Coffee`
- CTA: `Explore the Blend`

## Freshness Panel Copy

`Locked`:

- one oversized freshness message
- one short supporting paragraph

`Preferred` messaging focus:

- freshness
- aroma
- craft

Avoid:

- origin-heavy storytelling for v1
- long-form educational copy
- ecommerce detail blocks

## Metrics And Supporting UI

`Preferred`:

- if metrics are used, keep them short and decorative
- no more than 2 to 3 small values in the hero
- they should support the premium feel, not become data cards

## Copy Length Rules

`Locked`:

- headline: ideally 3 to 6 words
- supporting paragraph: short and readable at a glance
- CTA: 2 to 4 words
- navigation labels: single words where possible
